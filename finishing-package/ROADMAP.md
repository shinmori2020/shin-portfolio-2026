このフォルダには仕上げ4件の統合指示書が入っている。
**実行は必ず1件ずつ**(1 HANDOFF = 1セッション。/clear してから投げる)。
各HANDOFF完了後に該当サブフォルダを削除し 全件完了後にこのフォルダごと削除する。

## ③ 実画像の投入 — 完了記録(2026-07)

撮影・掲載まわりは完了。**残るは各作品の成果データ(outcomes)のみ**。

実施内容:

- 4作品の `cover`(ファーストビュー) と `full`(縦長全景) を Playwright で撮影
- `works.ts` の url をプレースホルダーから実ドメインへ差し替え 外部リンク用に `href` を追加
- 詳細ヒーローに「サイトを見る →」を設置(別タブ / rel=noopener noreferrer)
- ライトボックスを full 表示に配線。full が無い案件は「全体を見る」導線ごと非表示
- 掲載画像を WebP 化し **合計 14.7MB → 1.9MB(-87%)** に削減

撮影で踏んだ罠(reveal未発火の白紙 / 背景アニメの凍結 / 初期モーダル / npm のフラグ)と
その対処は **`docs/works-capture.md`** に集約。撮り直す前に必ず読むこと。

### 残作業: 成果データ(outcomes)

各作品の数値実績は**人間しか用意できない**ため未入力。未入力の間は詳細ページの
05「成果」セクションごと非表示になる(壊れないので急がなくてよい)。

用意でき次第 `src/data/works.ts` の各作品に `outcomes` を記入するだけで表示される。
書き方の記入例は works.ts 内にコメントで残してある(数値改善は before→after /
単発の実測値は after のみ / 定性成果は label+note)。

## 共通ルール(全HANDOFFに適用)

- 方針確認は省略し 実装→セルフ検証→各HANDOFF指定フォーマットで報告まで一括
- 迷ったら止まらず暫定判断で進め 報告の「判断した事項」に記載
- サイトの文体規約: **読点「、」を使わない**。半角スペースまたは文の分割で処理
- 優先順位: 速度(LCP/INP/CLS) > 既存機能の無事 > 世界観(引き算) > 見た目の忠実度
- tsc --noEmit / lint / next build 通過が全件の完了条件
